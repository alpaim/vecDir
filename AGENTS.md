# AGENTS.md

## Build & Test Commands

### Rust backend (run from `src-tauri/`)
- Build: `cargo build`
- Test: `cargo test`
- Lint: `cargo clippy --all-targets --all-features`
- Format check: `cargo fmt --check`
- Format fix: `cargo fmt`

### Frontend (run from project root)
- Lint: `bun run lint`
- Lint fix: `bun run lint:fix`
- Typecheck + build: `bun run build`
- Dev: `bun run dev`

## Architecture Overview

Tauri 2 desktop app for semantic file search.
- **Rust backend** (`src-tauri/src/`): SQLite via `sqlx`, vector search via `sqlite-vec` (768-dim cosine), embedding backends dispatched by `EmbeddingBackendType` enum.
- **Frontend** (`src/`): React + TanStack Router/Forms, TypeScript bindings auto-generated by `tauri-specta` into `src/lib/vecdir/bindings.ts`.
- Embedding backends: `OpenAICompat` (LLM describes image then embeds text), `VecBox` (OpenAI-style multimodal `/v1/embeddings`), `LlamaCpp` (llama.cpp `prompt_string` + `multimodal_data` format).

## Key Files
- `src-tauri/src/ai.rs` — AI client (OpenAI-compatible), shared helpers (image_to_base64, prepare_matroshka)
- `src-tauri/src/ai/vecbox.rs` — VecBox embedding client
- `src-tauri/src/ai/llamacpp.rs` — llama.cpp multimodal embedding client
- `src-tauri/src/database/models.rs` — data models including `EmbeddingConfig`, `EmbeddingBackendType`
- `src-tauri/src/indexer/processor.rs` — file processing dispatch by backend
- `src-tauri/src/search/embedding.rs` — query embedding dispatch by backend
- `src/components/settings/createSpace.tsx` — space creation form
- `src/components/settings/settings.tsx` — space edit form

## Conventions
- No comments in code unless explicitly requested.
- Rust: follow existing style (anyhow for errors, serde for (de)serialization, reqwest for direct HTTP).
- Frontend: TanStack Form, shadcn/ui components, lucide-react icons.
- `bindings.ts` is auto-generated — do not edit manually. Regenerated on debug build via tauri-specta.
